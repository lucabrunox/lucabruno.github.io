Smalltalk YX toward a new interpreter

Hello, Smalltalk YX is currently working on refactoring a part of the interpreter, relative to contexts and stack management. The current version uses one stack per process but always create contexts for each method or block entered. The new branch, called newinterp, won’t create those contexts anymore but they will be created only on-demand (e.g. thisContext). This means both less resources and garbage collector usage, and a possible speedup. More informations on the mailing list here.
